# Code of Conduct

## Our Standard

OM Core is a technical project. The goal is to create a community where people
can discuss modeling, software design, bugs, documentation, and implementation
details productively.

Participants are expected to be respectful, direct, and constructive.

## Expected Behavior

Examples of expected behavior include:

- focusing criticism on ideas, code, documentation, and design
- assuming good faith where reasonable
- giving actionable feedback
- being clear about uncertainty
- respecting different levels of experience
- helping keep discussions on topic
- accepting maintainer decisions even when you disagree

## Unacceptable Behavior

Examples of unacceptable behavior include:

- harassment, threats, or personal attacks
- discriminatory or degrading comments
- sexualized language or imagery
- sustained off-topic disruption
- trolling or deliberate provocation
- publishing private information without permission
- impersonation or misleading representation
- abusive behavior toward maintainers, contributors, or users

## Scope

This code of conduct applies to project spaces, including:

- GitHub issues and pull requests
- project discussions
- documentation contributions
- community chat spaces operated for OM Core
- events or calls organized around OM Core

It may also apply when behavior outside project spaces materially affects the
safety, trust, or operation of the project community.

## Enforcement

Maintainers may take action to protect the project and its participants. Actions
may include:

- asking a participant to stop specific behavior
- editing or removing comments
- closing issues or pull requests
- temporary or permanent restriction from project spaces
- refusing future contributions

Maintainers are expected to apply enforcement in a way that is proportionate to
the situation.
